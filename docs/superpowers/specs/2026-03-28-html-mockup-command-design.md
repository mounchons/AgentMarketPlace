# HTML Mockup Command Design Spec

> Date: 2026-03-28
> Plugin: ui-mockup v1.4.0 -> v1.5.0
> Author: Mounchons + Claude

## Summary

Add `/create-html-mockup` command to the ui-mockup plugin. This command creates browser-viewable HTML mockup pages using a shared `<master-page>` Web Component for consistent layout (sidebar, topbar, profile dropdown). Navigation data is sourced from Graph Brain. The frontend-design skill is invoked to produce production-grade HTML with Tailwind CDN.

## Goals

1. Create HTML mockups that can be opened directly in a browser
2. Use Web Components to provide a shared Master Page layout across all mockup pages
3. Pull navigation structure, topbar config, and profile dropdown from brain knowledge
4. Invoke frontend-design skill for high-quality visual output
5. Keep existing ASCII mockup workflow unchanged (additive, not replacement)

## Non-Goals

- Replacing the existing `/create-mockup` ASCII workflow
- Adding a dev server or build step
- Creating interactive/functional prototypes (these are visual mockups only)
- Supporting multiple master page variants per project

## Master Page Web Component

### Responsibilities

- Renders sidebar with nested navigation menu (collapsible, mobile overlay)
- Renders topbar with breadcrumb, sidebar toggle button, property/project name
- Renders profile dropdown with user avatar (initials), role label, menu items, logout
- Provides a slot for page content
- Highlights active menu item via `active-menu` attribute
- Responsive layout: desktop sidebar + mobile hamburger overlay

### Usage in Mockup Pages

```html
<!DOCTYPE html>
<html lang="th">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>User List - HTML Mockup</title>
  <script src="https://cdn.tailwindcss.com"></script>
  <link rel="stylesheet" href="master-page.css">
  <script src="master-page.js" defer></script>
</head>
<body>
  <master-page active-menu="users" page-title="User List">
    <!-- page content generated by frontend-design skill -->
  </master-page>
</body>
</html>
```

### Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `active-menu` | string | Menu item label or ID to highlight in sidebar |
| `page-title` | string | Page title shown in topbar breadcrumb |

### Data Source (Brain)

Query brain for:
1. **Navigation structure** -- sections, items, children, icons, hrefs, roles
2. **Topbar config** -- project/property name, breadcrumb style
3. **Profile dropdown** -- user display name, role labels, menu items (profile, settings, logout)
4. **Design tokens** -- colors (primary, sidebar-bg, etc.), fonts, spacing, radius

If brain has no data for a project, fall back to a sensible default template based on mockup_list.json entities/pages.

## Command Flow: `/create-html-mockup`

### Step 0: Read CLAUDE.md
Extract project conventions (same as /create-mockup):
- Alert library (SweetAlert2 default)
- URL convention (flat vs nested)
- Framework parameter syntax
- Component library
- CSS framework
- Design tokens

### Step 1: Read mockup_list.json
- If exists: show pending pages for selection
- If not exists: prompt user to run `/init-mockup` or specify page name

### Step 2: Query Brain for Navigation Data
Use brain search/get-knowledge tools to retrieve:
- Nav template / navigation structure
- Topbar configuration
- Profile nav dropdown structure
- Design tokens (if stored in brain)

### Step 3: Generate Master Page (if not exists)
If `.mockups/html/master-page.js` does not exist:
1. Use brain data to populate navigation items
2. Use design tokens from brain/CLAUDE.md for styling
3. Generate `master-page.js` (Web Component with Shadow DOM)
4. Generate `master-page.css` (shared styles, CSS custom properties)

If master page already exists, reuse it. Offer to regenerate if brain data has changed.

### Step 4: Generate HTML Mockup Page
1. Invoke frontend-design skill for the page content
2. Provide context: page info from mockup_list.json, CRUD type, entity fields, API endpoints
3. Use Tailwind CDN for utility classes
4. Wrap content in `<master-page>` tag with active-menu and page-title attributes
5. Include proper head section with meta tags, title, script/link references

### Step 5: Update Index Page
- Generate/update `.mockups/html/index.html`
- List all HTML mockup pages with links, page name, URL, CRUD type, status
- Style with Tailwind CDN for clean presentation

### Step 6: Update mockup_list.json
- Add `html_mockup_file` field to the page entry
- Update status if needed

## Master Page Template Structure (master-page.js)

The Web Component uses Shadow DOM with a slot element for page content.
Navigation data is embedded as a JSON array during generation from brain data.
The component handles:
- Sidebar rendering with nested menu items and active highlighting
- Topbar with breadcrumb from page-title attribute
- Profile dropdown with click-to-toggle behavior
- Sidebar toggle (collapse on desktop, overlay on mobile)
- Responsive breakpoints (desktop: fixed sidebar, mobile: hidden sidebar with hamburger)
- Close mobile sidebar on outside click or resize to desktop

### DOM Construction

The component uses DOM API methods (createElement, appendChild, textContent) for safe
DOM construction instead of string-based HTML insertion. This avoids XSS risks and
follows Web Component best practices.

### CSS Strategy

master-page.css defines CSS custom properties (design tokens) on :root.
Shadow DOM styles are loaded via a link element inside the shadow root.
This allows consistent theming while maintaining encapsulation.

## Integration Points

- **Graph Brain**: query for nav structure, topbar, profile dropdown, design tokens
- **frontend-design skill**: invoked to generate high-quality page content
- **mockup_list.json**: read page info, update with html_mockup_file path
- **CLAUDE.md**: extract project conventions and design tokens
- **Existing /create-mockup**: unchanged, both workflows coexist

## Edge Cases

1. **Brain has no nav data**: Fall back to generating nav from mockup_list.json pages/entities
2. **Master page already exists**: Reuse; offer regeneration option via argument `--regenerate-master`
3. **No mockup_list.json**: Allow specifying page name directly (same as /create-mockup behavior)
4. **Project has no CLAUDE.md**: Use sensible defaults (Tailwind, SweetAlert2, flat URLs)
5. **file:// protocol limitations**: Avoid ES modules in master-page.js; use classic script pattern so HTML files work when opened directly from filesystem
