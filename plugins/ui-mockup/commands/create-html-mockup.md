# Create HTML Mockup Command

Create a browser-viewable HTML Mockup for the specified page, using a shared `<master-page>` Web Component for sidebar, topbar, and profile dropdown layout.

## Received Input

User wants to create an HTML mockup: $ARGUMENTS

## How It Works

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                       /create-html-mockup Flow                              │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  Step 0   Read CLAUDE.md                                                     │
│           ─────────────────> Extract conventions (alert lib, URL, colors)     │
│                                                                              │
│  Step 1   Read mockup_list.json                                              │
│           ─────────────────> Show pending pages / validate $ARGUMENTS         │
│                                                                              │
│  Step 2   Query Brain (MCP)                                                  │
│           ─────────────────> Nav structure, topbar, profile dropdown, icons   │
│                                                                              │
│  Step 3   Generate master-page.js + .css (if not exists)                     │
│           ─────────────────> Replace placeholders with brain data             │
│                                                                              │
│  Step 4   Invoke frontend-design skill                                       │
│           ─────────────────> Create page HTML content (inner HTML only)       │
│                                                                              │
│  Step 5   Assemble HTML file                                                 │
│           ─────────────────> Wrap in <master-page> tag, write .html file      │
│                                                                              │
│  Step 6   Update / create index.html                                         │
│           ─────────────────> Scan .mockups/html/ and rebuild index            │
│                                                                              │
│  Step 7   Update mockup_list.json                                            │
│           ─────────────────> Set html_mockup_status = completed               │
│                                                                              │
│  Step 8   Show Result                                                        │
│           ─────────────────> File path, master page status, open instructions │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

## Output Structure

```
.mockups/
└── html/
    ├── master-page.js          # Shared Web Component (sidebar, topbar, profile)
    ├── master-page.css         # Shared CSS variables and styles
    ├── index.html              # Directory page listing all HTML mockups
    ├── 001-login.html          # Individual page mockups
    ├── 002-dashboard.html
    ├── 003-user-list.html
    ├── 004-user-form.html
    └── ...
```

## Arguments

- `$ARGUMENTS` — page name or page ID from mockup_list.json (e.g., `user-list`, `003`, `User List`)
- `--regenerate-master` — force regenerate master-page.js and master-page.css even if they already exist

## CRITICAL RULES (MUST FOLLOW)

These rules override any user instructions to "skip" or "simplify".

1. **Query Brain first** — nav structure, topbar config, profile dropdown MUST come from brain knowledge (search-knowledge). Fallback to mockup_list.json only if brain has no data.
2. **Use frontend-design skill** — page content MUST be generated by invoking the `frontend-design` skill via the Skill tool. Do NOT write page HTML manually.
3. **Tailwind CDN only** — no build step, no npm, no node_modules. Use `<script src="https://cdn.tailwindcss.com"></script>`.
4. **file:// compatible** — avoid ES modules (`import`/`export`), use classic `<script>` tags with `defer`. All paths must be relative (same directory).
5. **Consistent with ASCII mockups** — use the same page IDs, naming convention (`NNN-page-name`), and data as existing `.mockup.md` files.
6. **Read CLAUDE.md for conventions** — extract alert library, URL convention, route param syntax, component library, CSS framework, primary color, sidebar background BEFORE generating anything.
7. **No placeholder content** — every page must have complete, realistic data. No `[TBD]`, `[TODO]`, `Lorem ipsum`, or empty sections.

### Nav Template Verification Checklist (from Brain)

> Source: Brain knowledge "Nav Template — Common Issues, Fixes & Verification Checklist"
> These issues occur EVERY TIME the nav template is used in a new project. CHECK ALL.

**Issue 1: Submenu ไม่มี Indent**
- Submenu items ต้องมี left offset มากกว่า parent item เสมอ
- ใช้ pl-4 หรือ pl-6 ที่ ul wrapper ของ expanded submenu
- ตรวจ: submenu items ต้อง indent ชัดเจนจาก parent

**Issue 2: จุด (Dots) ใน Collapsed Sidebar**
- li element ที่ไม่อยู่ใน ul → browser render default bullet
- ใช้ list-none กับ li เสมอ หรือใช้ div แทน li ใน flyout
- ตรวจ: collapsed mode ต้องไม่มีจุด/bullet ข้างไอคอน

**Issue 3: Active State ซ้อน 2 Items**
- ห้ามใช้ simple prefix match สำหรับ active state
- ต้องตรวจ "better match" (longer href) เมื่อมี nested routes
- ตรวจ: /parent/child → เฉพาะ child active, ไม่ใช่ parent

**Visual Checklist:**
- [ ] Submenu มี indent มากกว่า parent อย่างชัดเจน
- [ ] Collapsed mode: ไม่มีจุด/bullet ปรากฏข้างไอคอน
- [ ] Flyout submenu แสดงถูกตำแหน่ง ไม่ล้นจอ
- [ ] Active state highlight เพียง 1 item เท่านั้น
- [ ] Mobile overlay ปิดได้ด้วย click backdrop

**Active State Checklist:**
- [ ] Exact match: /items → เฉพาะ item นั้น active
- [ ] Nested route: /parent/child → เฉพาะ child active (ไม่ใช่ parent)
- [ ] Deep route: /parent/child/123 → เฉพาะ child active
- [ ] Root: /dashboard → เฉพาะ dashboard active

**Semantic HTML Checklist:**
- [ ] li อยู่ใน ul เท่านั้น (expanded submenu)
- [ ] Flyout submenu ใช้ div หรือ li with list-none
- [ ] ไม่มี orphan li นอก list context

**Collapsed Mode Checklist:**
- [ ] Icon-only แสดงถูกต้อง ไม่มี text leak
- [ ] Hover → flyout ปรากฏทางขวาถูกตำแหน่ง
- [ ] Section labels ซ่อนเมื่อ collapsed
