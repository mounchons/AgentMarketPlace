## ตัวอย่าง Persona

### ป้าสมศรี (End User — ทักษะ IT ต่ำ)

#### ข้อมูลพื้นฐาน
- **ชื่อ**: ป้าสมศรี
- **อายุ**: 58
- **อาชีพ**: แม่ค้าตลาด
- **บทบาทในระบบ**: ผู้ใช้ทั่วไป

#### อุปกรณ์และสภาพแวดล้อม
- **อุปกรณ์หลัก**: มือถือ Android เก่า (Samsung Galaxy A10)
- **ขนาดจอ**: 6.2" (จอเล็ก)
- **ระบบปฏิบัติการ**: Android 11
- **Browser**: Chrome (version เก่า)
- **ความเร็วเน็ต**: ช้า (3G / WiFi ตลาด)

#### ทักษะและประสบการณ์
- **ทักษะ IT**: ต่ำ — ใช้แค่ LINE กับ Facebook
- **ประสบการณ์**: ไม่เคยใช้ระบบออนไลน์สั่งซื้อ
- **ภาษา**: ไทย (อ่านภาษาอังกฤษไม่ออก)

#### เป้าหมายและแรงจูงใจ
- **เป้าหมายหลัก**: สั่งของมาขายต่อ สั่งทีละเยอะ
- **ความถี่**: สัปดาห์ละ 2-3 ครั้ง
- **แรงจูงใจ**: ลูกบอกว่าสั่งออนไลน์ถูกกว่าไปซื้อเอง

#### Pain Points
- อ่านตัวเล็กไม่ชัด
- กลัวกดผิดแล้วเสียเงิน
- ไม่เข้าใจภาษาอังกฤษในปุ่มต่างๆ

#### พฤติกรรมการใช้งาน
- ถามลูกช่วยกดเมื่อไม่แน่ใจ
- Screenshot ทุกขั้นตอนเพื่อดูทีหลัง
- กดซ้ำเมื่อคิดว่าระบบไม่ตอบสนอง

### บอส ธนพล (Power User — ทักษะ IT สูง)

#### ข้อมูลพื้นฐาน
- **ชื่อ**: บอส ธนพล
- **อายุ**: 32
- **อาชีพ**: เจ้าของ SME
- **บทบาทในระบบ**: Business user / Admin

#### อุปกรณ์และสภาพแวดล้อม
- **อุปกรณ์หลัก**: iPhone 16 + MacBook Pro
- **ขนาดจอ**: 6.7" (มือถือ) / 16" (laptop)
- **ระบบปฏิบัติการ**: iOS 19 / macOS
- **Browser**: Safari + Chrome
- **ความเร็วเน็ต**: เร็ว (5G / WiFi ออฟฟิศ)

#### ทักษะและประสบการณ์
- **ทักษะ IT**: สูง — ใช้หลายแพลตฟอร์ม (Shopee, Lazada, Line OA)
- **ประสบการณ์**: ใช้ระบบ e-commerce มากกว่า 5 ปี
- **ภาษา**: ไทย + อังกฤษ

#### เป้าหมายและแรงจูงใจ
- **เป้าหมายหลัก**: สั่งซื้อ bulk, ต้องการ invoice/ใบกำกับภาษี, track orders
- **ความถี่**: ทุกวัน
- **แรงจูงใจ**: ประหยัดเวลา จัดการทุกอย่างจากที่เดียว

#### Pain Points
- ไม่มีเวลา ต้องเร็ว
- ต้อง track หลาย orders พร้อมกัน
- ต้องการ export ข้อมูลสำหรับบัญชี

#### พฤติกรรมการใช้งาน
- เปิดหลาย tab ทำหลายอย่างพร้อมกัน
- ใช้ keyboard shortcuts เมื่อทำได้
- เปรียบเทียบราคาหลายร้านก่อนสั่ง

## หลักการสร้าง Persona ที่ดี

1. **หลากหลาย** — ต้องมีทั้ง IT ต่ำ, กลาง, สูง
2. **สมจริง** — ใช้ชื่อไทย, อาชีพจริง, อุปกรณ์จริง
3. **มี Pain Points ชัด** — แต่ละคนมีปัญหาต่างกัน
4. **ครอบคลุม roles** — ถ้าระบบมี Admin, User, Guest ต้องมี persona แต่ละ role
5. **เสริมด้วย internet research** — ใช้ข้อมูลจริงจากภายนอกเพื่อเพิ่ม pain points
